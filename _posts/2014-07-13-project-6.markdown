description: Data analytics and user testing revealed that after booking a restaurant many people simply want to change one or two details about their reservation. Rather than cancelling the booking and starting the whole process again, a shortened edit booking flow might provide a much better user experience. In order to implement this feature I first wrote pure Ruby which made requests to the company's internal SOAP XML API. I then ported over the core Ruby logic into a fully-fledged Rails app with views, models and controllers.
